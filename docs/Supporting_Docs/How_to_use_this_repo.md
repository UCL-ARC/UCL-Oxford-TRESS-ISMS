# Use of the Research Data ISMS GitHub Repository

## Issues
Issues are recorded in the 
[Dev Team Project](https://github.com/orgs/UCL-ARC/projects/85)

## Issue Types and Labels
Issue types and tags can be used to group issues

### Issues Type - Risk
Risks against the ISMS are recorded in the repo in accordance with the [Risk Assessment and Treatment Policy](../Controlled_Documents/RISM04-Risk_Assessment_and_Treatment_Procedure.md). This includes the requirement for an impact and likelihood score and assigned owner where this has been determined.

### Label - OMG Action
OMG actions are stored in the ISMS Dev Team project and labelled "OMG Action". These actions can be filtered and used to track actions for the OMG.

## Change Management
Changes to the controlled documents in the repository are made via pull requests. Changes should only be committed to main if they are made using the following procedure:

1. Proposer creates a branch (for example from main).
2. Make changes to document on that branch.
3. Once changes are complete, create a Pull Request (PR). All PRs should be named %usersname%-%PRName%. The PR should include a link to the issue that is being addressed through the PR (as per template).
4. PR is reviewed by another member of the team (for Standard Changes) or by the OMG/IRGC (for Normal Changes).
5. After review, if the PR is approved the proposer merges the changes into main.
6. Delete the branch

## Public Mirror
There is a public mirror of the controlled documents at the following URL. This is published each time there is a PR merged into main.

https://isms.arc.ucl.ac.uk/

## Access Control
| Group  |    Permission                                          |

| Information Governance Team | Admin/Read permissions to make changes (following the above procedure)|
| All other in scope roles | Read permission to Controlled Documents via https://isms.arc.ucl.ac.uk/|
