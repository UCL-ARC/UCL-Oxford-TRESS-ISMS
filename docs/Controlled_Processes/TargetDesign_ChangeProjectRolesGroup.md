# Overview
This process allows Information Asset OWners or Information Asset Administrators to modify the Project Roles Group of a Project that they own. This process is owned by the ARC TRE Envrionment Owner.

## 5 Safes Link
**Safe Projects**: project owners are able to modify the assigned role(s) or overall memebrship of the Project Roles Group to reflect changes in project personel or role.

# Process
## Purpose
Process ensures Information Asset Owners are able to modify previously assigned Role and overall memebrship of the Project Roles Group to reflect changes in personnel, training status or role. This allows them to have decision-making ability over all other members of the Project in terms of controlling the use of sensitive or confidential information.

## Scope

All changes to the Prjoect Roles Group of any existing UCL TRE Project falls within the scope of the ISMS and this process.

## Responsibilities
- The Information Asset Owner/Administrator: requests changes to the Project Roles Group in line with their resopnsibility to properly manage the personnel within the Studies and Projects they own, which is a responsibility they agree to in the Study Agreement. 
- TRE Environment Team: implements into the technical Environment all valid requests made by Information Asset Owners in a timely manner.

## Prerequisites

- The Project Roles Group to which changes are to be made must already exist.
- The parent Study of the Project must be owned by the Information Asset Owner or Administrator making the change request

## Steps
1. The Information Asset Owner/Administrator communicates their desired changes to the Project Roles Group via the Service Management front end.
2. If the change requires new users or a change in IAO/IAA, the respective users copmlete the necessary onboarding tasks, notifying the TRE Environment Team once tasks have been completed.
3. If the change affects the previously agreed billing/payment arrangement, TRE Environment Team discuss and agree a new payment arrangement with the IAO/IAA.
4. TRE Environment Team validates the request by:
  - confirming the request has in fact come from the Information Asset Owner or Administrator of the Study that the Project resides within
  - ensuring the requested change to the Project Roles Group is technically supported
5. TRE Environment Team updates the Project Roles Group definition and deployes the updated configuration
